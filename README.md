# מעקב כספים לערב פוקר — גרסת HTML סטטית

זו גרסה חלופית של האפליקציה: **קובץ HTML אחד בלבד**, בלי React, בלי Vite, בלי `npm install`, בלי שלב build. אפשר להעלות אותו ל-GitHub Pages (או כל אחסון סטטי אחר) ולהריץ אותו ישירות בדפדפן.

Supabase עדיין משמש כמקור האמת היחיד (בדיוק כמו בגרסת ה-React) — כל השינויים מסונכרנים בזמן אמת בין כל מי שפותח את אותו קישור.

## הקמה מהירה

### 1. Supabase (בדיוק כמו בגרסה הקודמת)

1. צרו פרויקט ב-[supabase.com](https://supabase.com).
2. **SQL Editor** → **New query** → הדביקו את כל התוכן של [`supabase/schema.sql`](./supabase/schema.sql) → **Run**.
3. **Project Settings** → **API** → העתיקו את ה-**Project URL** וה-**anon public key**.

### 2. הגדרת החיבור

פתחו את `index.html` בעורך טקסט, וחפשו את השורות האלה קרוב לתחילת ה-`<script>`:

```js
const SUPABASE_URL = 'https://your-project-ref.supabase.co';
const SUPABASE_ANON_KEY = 'your-anon-public-key';
```

והחליפו אותן בערכים האמיתיים שלכם מהשלב הקודם. שימו לב: יש להשתמש רק במפתח ה-**anon public**, לעולם לא ב-service-role key (הוא לא אמור להופיע כאן בכלל).

### 3. הרצה

**הכי פשוט — GitHub Pages:**
1. צרו repository חדש ב-GitHub, והעלו אליו את `index.html` (אפשר גם בלי התיקייה `supabase/`, היא רק לתיעוד).
2. **Settings** → **Pages** → Source: **Deploy from a branch** → בחרו את ה-branch הראשי והתיקייה `/ (root)`.
3. אחרי דקה-שתיים תקבלו קישור כמו `https://your-username.github.io/your-repo/` — זה הקישור לשתף עם קבוצת הפוקר.

**מקומית (לבדיקה):** אפשר גם פשוט לפתוח את `index.html` ישירות בדפדפן, או להריץ שרת סטטי מקומי כמו `npx serve .`.

## איך זה עובד

- **ערבי פוקר** — רשימה כרונולוגית של כל הערבים (כולל ישנים), עם אפשרות ליצור ערב חדש, לפתוח ערב קיים ולראות/להמשיך לערוך את הפרטים שלו, ולמחוק ערב (עם אישור כפול).
- **שחקנים קבועים** — רשימה משותפת של שחקנים לבחירה מהירה בכל ערב.
- **מסך ערב** — סיכום קופות (משחק/בונטי/מזומן/אשראי עם פירוט קניות ראשוניות מול ריביי), כרטיס לכל שחקן עם רשימת קניות ממוספרת וניתנת לעריכה (סכום + מזומן/אשראי), הוספת ריביי, הערות, תנועות (כולל ביטול/שחזור), ואזור בונטי עם הגדרת פרסים (כולל חלוקה אוטומטית בכפולות של 50), חלוקת פרס לכל שחקן, ויתרת קופה מתעדכנת.
- כל פעולה כספית נשמרת כתנועה נפרדת בטבלת `transactions`, בדיוק כמו בגרסת ה-React — שום סכום מצטבר לא נשמר ידנית.

## הבדלים מגרסת ה-React

- אין שלב build, אין תלות ב-Node/npm — קובץ אחד שרץ ישירות בדפדפן.
- הקוד כתוב ב-JavaScript רגיל (vanilla) במקום React/TypeScript.
- אין מסך "סיכום ערב" נפרד ואין הבחנה בין ערב "פעיל" ל"הושלם" — כל הערבים (כולל ישנים) פשוט מופיעים ברשימה ואפשר לפתוח ולראות/לערוך את הפרטים שלהם באותו מסך.
- העיצוב זהה (ירוק שולחן פוקר, זהב, RTL בעברית, מותאם למובייל), רק ממומש ב-CSS רגיל במקום Tailwind.

## הערת אבטחה

מדיניות ה-Row Level Security מאפשרת לכל אחד (כולל אנונימי) לקרוא ולכתוב לכל הטבלאות, בהתאם לדרישת "אין התחברות, כולם רואים ועורכים יחד". מכיוון שזהו קובץ סטטי, ה-URL וה-anon key גלויים לכל מי שצופה בקוד המקור של הדף — זה תקין ומוכר בעולם Supabase (המפתח הזה מיועד לשימוש בצד הלקוח), אבל חשוב לוודא **שלעולם לא** מכניסים לכאן את ה-service-role key.
